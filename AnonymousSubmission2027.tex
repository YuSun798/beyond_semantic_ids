\begin{abstract}
Generative recommendation (GR) has recently emerged as a powerful paradigm, retrieving items by autoregressively decoding discrete Semantic IDs (SIDs) over learned codebooks. While this paradigm has demonstrated competitive performance, it fundamentally relies on beam search for inference---a strategy with known theoretical constraints. In this paper, we identify and characterize a structural bottleneck in SID-based models: the Beam Search Reachability Ceiling. We provide empirical and theoretical evidence that tree-structured autoregressive generation imposes rigid correlation constraints, which inherently limit the model's ability to represent arbitrary item distributions. Specifically, we quantify that the reachability of tail or sparse items is up to three times lower than that of popular items in standard SID frameworks. To systematically investigate this phenomenon, we propose CQG-Rec, a minimal vocabulary-free analytical probe that replaces discrete token generation with continuous query generation in a stable, frozen embedding space. Through controlled experiments across multiple standard benchmarks, we reveal a distinct ``Crossover Phenomenon'': while SID-based models excel at extremely shallow top-K retrieval, CQG-Rec consistently outperforms them in deep retrieval and long-tail coverage without the need for codebook training or complex beam search. Our findings establish that the reachability ceiling is a structural property of the decoding mechanism itself rather than an implementation artifact. This study challenges the necessity of vocabulary infrastructure in generative recommendation and points toward continuous representation as a more robust path for scaling to massive, dynamic catalogs.
\end{abstract}

\noindent\textbf{TL;DR.} Identifying the Reachability Ceiling of Semantic IDs, we propose CQG-Rec, a vocabulary-free regime using continuous queries to enable retraining-free updates and superior long-tail retrieval via a Crossover Phenomenon.

\section{Introduction}

Generative recommendation retrieves items by generating symbolic identifiers rather than directly scoring the entire catalog. The most influential instance of this paradigm is Semantic-ID (SID) recommendation: items are first quantized into discrete code sequences, and an autoregressive model decodes those sequences conditioned on user history \citep{rajput2023tiger,zheng2024letter,wang2024eager}. This formulation is attractive because it converts recommendation into a familiar sequence-generation problem. It also supports constrained decoding through a trie, ensuring that the generated token sequence corresponds to a valid item.

However, the same discrete infrastructure also creates a structural bottleneck. SID systems depend on beam search at inference time. Beam search explores only a prefix-limited subset of the exponentially large code tree, and recent theoretical work shows that constrained beam decoding has inherent reachability limits in generative retrieval \citep{wu2025beam,hou2026expressiveness}. In other words, an item can have a valid SID and still receive zero retrieval probability if its path never enters the beam. This distinction is important: code assignment and code retrieval are different problems.

This paper asks a concrete question: \emph{how does the beam-search reachability ceiling manifest in standard recommendation benchmarks?} Aggregate Recall@K does not answer this question, because it mixes popular and rare items. We therefore measure reachability at the item level and stratify by training frequency. The result is striking. On ML-1M, a TIGER-style SID model with beam width 100 reaches 70.8\% of popular items but only 24.0\% of rare items, a 2.95$\times$ gap. At the same time, code collisions are negligible: 99.4\% of items receive unique codes. The bottleneck is thus not mainly codebook capacity; it is inference-side exploration.

This popularity-dependent ceiling creates a second empirical pattern that we call the \emph{Crossover Phenomenon}. At very shallow retrieval depth, SID models can still perform well because autoregressive decoding concentrates probability mass on high-confidence paths. On rare items in ML-1M, TIGER outperforms a continuous alternative at R@10. But as K increases, the beam exhausts its reachable set. A vocabulary-free continuous retriever continues to retrieve from the full catalog and overtakes SID retrieval at deeper K. On the rarest quartile, TIGER and the continuous model tie at R@90 = 0.2396, exactly matching TIGER's rare-item reachability ceiling.

To isolate this mechanism, we propose \textbf{CQG-Rec} (Continuous Query Generation for Recommendation), a deliberately minimal vocabulary-free analytical probe. CQG-Rec does not use item tokens, codebooks, or beam search. Instead, it learns a small causal Transformer that maps a user's interaction history into a continuous query vector in a frozen item embedding space. Retrieval is performed by dense scoring or approximate nearest neighbor (ANN) search. This design is intentionally simple: it is not meant to win through architectural complexity, but to test whether discrete vocabulary infrastructure is necessary for generative retrieval.

Our contributions are:
\begin{itemize}
    \item \textbf{Popularity-dependent reachability ceiling.} We empirically quantify how beam-search reachability varies across the item popularity spectrum. Rare items are nearly three times less reachable than popular items under a standard SID framework.
    \item \textbf{Crossover Phenomenon.} We identify a depth-dependent tradeoff: SID decoding can win at shallow top-K precision, but continuous retrieval overtakes at deeper K as the beam-search ceiling binds.
    \item \textbf{Vocabulary-free analytical probe.} We introduce CQG-Rec, a minimal continuous-query recommender that removes codebook training, item tokenization, beam search, and vocabulary maintenance.
    \item \textbf{Controlled empirical study.} Across ML-1M, Amazon Beauty, and Amazon Sports, we analyze retrieval quality, reranked performance, loss functions, LLM backbone failures, tokenization instability, and scalability.
\end{itemize}

\section{Related Work}

\textbf{Semantic-ID generative recommendation.}
TIGER \citep{rajput2023tiger} introduced hierarchical Semantic IDs using residual vector quantization and autoregressive decoding. Subsequent work improves code learning, alignment, and model design: LETTER integrates collaborative semantics \citep{zheng2024letter}, EAGER introduces behavior-semantic collaboration \citep{wang2024eager}, IDGenRec studies textual IDs \citep{tan2024idgenrec}, DIGER proposes differentiable indexing \citep{fu2026diger}, and UniGRec explores soft identifiers \citep{li2026unigrec}. These methods differ in how identifiers are constructed and aligned, but they retain a discrete vocabulary interface and typically require constrained autoregressive decoding. Our work targets a shared property of this family: the reachability ceiling induced by beam search over a structured identifier space.

\textbf{Reachability and expressiveness limits.}
Theoretical work has begun to formalize limitations of tree-structured generative retrieval. \citet{wu2025beam} prove error lower bounds for constrained beam search, while \citet{hou2026expressiveness} show that autoregressive tree generation imposes correlation constraints that limit representable item distributions. These analyses establish the existence of a structural ceiling. We complement them by measuring the ceiling empirically, showing that it is strongly popularity-dependent and that it creates a depth-dependent crossover in retrieval quality.

\textbf{Popularity bias in SID systems.}
Recent work also studies training-side popularity bias. For example, tail tokens may receive fewer updates under maximum-likelihood training, motivating modified objectives or tokenization schemes. Our analysis is complementary: even when code assignment succeeds, code retrieval can fail. In our ML-1M experiments, 99.4\% of items receive unique codes, yet only 63.8\% are ever reachable under beam width 100. This separates training-side code quality from inference-side exploration.

\textbf{Vocabulary-free and continuous recommendation.}
Continuous alternatives avoid discrete item vocabularies. DiffuRec \citep{li2023diffurec}, DreamRec \citep{yang2023dreamrec}, and ContRec \citep{qu2026contrec} use diffusion or continuous latent generation. GPT4Rec \citep{li2023gpt4rec} generates textual queries for retrieval. CQG-Rec occupies a deliberately simpler point in the design space: it generates a dense query vector directly in a frozen item embedding space and retrieves with dense scoring or ANN. The main contribution is not that continuous retrieval exists, but that a minimal continuous probe reveals the SID reachability ceiling and the crossover behavior.

\textbf{Sequential and dual-encoder recommendation.}
Sequential recommendation models such as GRU4Rec \citep{hidasi2016gru4rec}, SASRec \citep{kang2018sasrec}, and BERT4Rec \citep{sun2019bert4rec} score candidate items from user histories. Dual-encoder retrieval originates from neural information retrieval \citep{huang2013dssm} and is widely deployed in large-scale recommendation \citep{covington2016youtube}. CQG-Rec inherits the dense retrieval interface but differs from standard two-tower systems in its experimental role: it is used as a controlled probe for the vocabulary and beam-search assumptions in generative recommendation.

\section{Method}

\subsection{Problem Formulation}

Let $\mathcal{I}$ be a catalog of $N$ items and let user $u$ have a chronological interaction sequence $s_u=(i_1,\ldots,i_t)$. The task is to retrieve the next item $i_{t+1}$. SID-based GR maps each item $i$ to a discrete code sequence $z_i=(z_{i,1},\ldots,z_{i,L})$ and trains an autoregressive model $p(z_i\mid s_u)$. Inference uses beam search over valid code prefixes.

CQG-Rec instead assumes a frozen item embedding matrix $E\in\mathbb{R}^{N\times d}$, where each row $e_i$ is an L2-normalized item vector. The model learns a query generator $f_\theta$:
\begin{equation}
q_u = f_\theta(e_{i_1}, e_{i_2}, \ldots, e_{i_t}), \quad q_u\in\mathbb{R}^d .
\end{equation}
The recommendation score is cosine similarity:
\begin{equation}
\operatorname{score}(u,i)=q_u^\top e_i .
\end{equation}
The top-K items are retrieved by exact dense scoring or ANN search. Every item in $E$ is reachable by construction.

\subsection{Continuous Query Generator}

The query generator is a compact causal Transformer. Each history item embedding $e_{i_j}\in\mathbb{R}^{64}$ is projected to hidden dimension $D=256$ and combined with a learned position embedding:
\begin{equation}
x_j = W_{\mathrm{in}} e_{i_j}+p_j .
\end{equation}
Four pre-norm causal self-attention blocks with four heads and feed-forward dimension 1024 process the sequence. The final hidden state is projected back into the item space:
\begin{equation}
q_u = W_{\mathrm{out}} h_t .
\end{equation}
The model has 3.2M trainable parameters, approximately 45\% fewer than the TIGER configuration used in our ML-1M comparison.

\subsection{Training Objectives}

We evaluate three objectives to expose the accuracy--scalability tradeoff.

\textbf{Full-catalog cross-entropy.}
Our primary objective treats retrieval as classification over all items:
\begin{equation}
\mathcal{L}_{\mathrm{CE}}
= -\frac{1}{B}\sum_{b=1}^{B}
\log
\frac{\exp(q_b^\top e_b^+/\tau)}
{\sum_{j=1}^{N}\exp(q_b^\top e_j/\tau)} ,
\end{equation}
where $\tau=0.07$. CE gives complete catalog visibility at training time but costs $O(N)$ per step.

\textbf{Bayesian personalized ranking.}
For larger catalogs, we use one positive and one sampled negative:
\begin{equation}
\mathcal{L}_{\mathrm{BPR}}
=-\frac{1}{B}\sum_{b=1}^{B}
\log \sigma(q_b^\top e_b^+ - q_b^\top e_b^-).
\end{equation}
BPR scales independently of $N$ but provides weaker training-time catalog coverage.

\textbf{Mean-squared error.}
We also test direct embedding regression:
\begin{equation}
\mathcal{L}_{\mathrm{MSE}}
=\frac{1}{B}\sum_{b=1}^{B}\|q_b-e_b^+\|_2^2 .
\end{equation}
MSE preserves geometric alignment but yields lower retrieval precision than CE in our experiments.

\subsection{Retrieval and Reranking}

Standalone CQG-Rec retrieves by ranking items according to $q_u^\top e_i$. For larger catalogs, exact scoring can be replaced with an ANN index such as FAISS \citep{johnson2021faiss}. We also evaluate a modular retrieve-and-rerank pipeline: CQG-Rec retrieves 100 candidates, then a pretrained SASRec model reranks only those candidates. This isolates whether CQG-Rec can provide a high-quality candidate set for conventional ranking systems.

\subsection{Contrast with SID Retrieval}

Figure~\ref{fig:pipeline} compares the two inference mechanisms. SID retrieval requires codebook learning, autoregressive decoding, beam search, and code-to-item lookup. Beam search creates a nonzero-probability set that is smaller than the catalog. CQG-Rec requires only a query generator and a dense index; every encoded item can be retrieved without adding new output tokens or retraining the model.

\textbf{Protocol.}
We use leave-one-out evaluation: the last item is held out for testing, the second-to-last item for validation, and earlier interactions for training. Historical items are excluded from ranking. We report Recall@K and NDCG@K, primarily for $K\in\{10,30,50,70,90\}$ in reachability analysis and $K\in\{5,10,20\}$ in cross-dataset evaluation.

We also include non-learned dense retrieval baselines: random retrieval, last-item kNN, and mean-of-5 kNN.

\subsection{LLM Backbone Investigation}

We also tested whether a pretrained LLM backbone improves continuous query generation. Seven Qwen2.5-7B configurations across three paradigms consumed 23 GPU-hours. The result was negative but informative. Shared dual-tower training collapsed item representations, with item-item cosine similarity increasing toward 0.9936. A frozen-item hybrid avoided item collapse but produced loss-metric decoupling: InfoNCE loss improved while Recall@10 remained only 18\% of the SASRec gate. A text-aware hybrid performed best, reaching R@10 = 0.2118 on ML-1M, but still trailed the 3.2M scratch Transformer by 12.7\%.

\begin{figure}[t]
\centering
\includegraphics[width=\columnwidth]{Figures/fig4_llm_diagnostic.pdf}
\caption{LLM backbone diagnostics. Pretrained language backbones do not automatically align with the frozen collaborative item space; the best text-aware Qwen hybrid still trails the scratch query Transformer.}
\label{fig:llm_diagnostic}
\end{figure}

This analysis does not imply that LLMs cannot be used for recommendation. Rather, it shows that pretrained language representations are not automatically aligned with a low-dimensional frozen collaborative item space. Continuous query generation requires explicit geometric grounding.

\subsection{Tokenization Instability}

The auxiliary experiments also expose fragility in discrete tokenization. Six neural RQ-VAE configurations collapsed to degenerate codebooks with less than 10\% utilization. A k-means fallback achieved high code utilization but showed large downstream variance depending on decoder configuration. This supports the broader argument: SID pipelines contain multiple coupled components---tokenizer, decoder, beam search, and lookup---and failures in any component can limit retrieval.

\subsection{Scalability}

CE-frozen requires full-catalog softmax and is therefore appropriate for catalogs up to roughly tens of thousands of items. BPR-frozen scales to larger catalogs and was used for the 112K-item Yelp setting, but it trades off accuracy. In production-scale systems, the natural next step is sampled softmax, cached ANN hard negatives, or ANN-accelerated contrastive training. Importantly, the inference interface remains scalable: once a query is generated, retrieval can use standard ANN infrastructure and new items can be inserted by adding their embeddings to the index.

\section{Discussion}

\textbf{What the ceiling means.}
The reachability ceiling is not the same as poor model accuracy. It is a support constraint: items outside the beam's explored code paths have zero chance of being retrieved, no matter how good a downstream reranker is. This is especially problematic in retrieve-and-rerank systems because missing candidates cannot be recovered later.

\textbf{Why the crossover matters.}
The Crossover Phenomenon explains why SID methods can look strong under shallow top-K metrics while still being structurally limited. At R@10, concentrated autoregressive probability can help. At deeper K, dynamic catalogs, or long-tail retrieval, the finite beam support becomes a bottleneck. CQG-Rec has the opposite profile: no reachability ceiling, but query quality depends on embedding alignment.

\textbf{Practical implications.}
For static catalogs with strong top-10 precision requirements, SID methods remain competitive. For massive or dynamic catalogs where coverage, tail retrieval, and retraining-free updates matter, continuous-query retrieval is a simpler and more robust interface. CQG-Rec can also coexist with existing rankers by serving as the recall stage.

\section{Limitations}

Our study has several limitations. First, the strongest reachability and crossover analysis is conducted on ML-1M; cross-dataset confirmation of per-quartile reachability at larger scale remains ongoing. Second, CE-frozen uses full-catalog softmax, which does not scale directly to million-item training without sampling or ANN-negative approximations. Third, the TIGER comparison uses a single seed in the current draft, while CQG-Rec CE uses three seeds. Fourth, CQG-Rec depends on the quality of the frozen item embedding space; if that space fails to encode preference-relevant distinctions, continuous retrieval will inherit the error. Fifth, pretrained LLM backbones underperformed in our tested setting, so the present method uses a scratch Transformer rather than a large pretrained generator. Finally, SID methods may recover part of the tail gap through improved training objectives or tokenization, although such changes cannot remove beam-search support constraints without changing the inference mechanism itself.

\section{Conclusion}

This paper identifies the Beam Search Reachability Ceiling as a practical bottleneck in SID-based generative recommendation. The ceiling is strongly popularity-dependent: rare items are up to three times less reachable than popular items. This structure induces a Crossover Phenomenon in which SID methods can win shallow precision, while vocabulary-free continuous retrieval catches up or surpasses them at deeper K and in long-tail coverage. CQG-Rec demonstrates that a minimal continuous-query generator in a frozen embedding space can remove codebooks, item vocabularies, and beam search while preserving strong recommendation quality. These findings challenge the assumption that Semantic IDs are necessary infrastructure for generative recommendation and suggest continuous representation as a scalable direction for dynamic catalogs.

\section{Experimental Protocol and Reproducibility}
\label{app:protocol}

\subsection{Dataset Splits}
\label{app:splits}

All datasets are converted into chronological user sequences. For each user, we use the last interaction for testing, the penultimate interaction for validation, and all earlier interactions for training. Users with insufficient history after filtering are removed. Evaluation is full-catalog unless otherwise stated: the target item is ranked against all candidate items rather than against sampled negatives. The reported top-$K$ metrics use $K\in\{5,10,20,30,50,70,90\}$ when available.

\subsection{Evaluation Metrics}
\label{app:metrics}

For a user $u$ with held-out target item $i_u^\star$, Recall@$K$ is one if $i_u^\star$ appears in the top-$K$ list and zero otherwise. NDCG@$K$ discounts a hit by the target rank. For SID models, we additionally report the beam-search reachable rate:
\[
\mathrm{Reachable@B} =
\frac{1}{|\mathcal{I}|}
\sum_{i\in\mathcal{I}}
\mathbf{1}\{i\in \cup_{u\in\mathcal{U}} \mathrm{Beam}_B(u)\},
\]
where $\mathrm{Beam}_B(u)$ is the set of items decoded by beam search with beam width $B$ for user $u$. This metric measures support coverage of the inference procedure, not ranking accuracy conditional on support.

\subsection{CQG-Rec Dataloader}
\label{app:cqg_dataloader}

CQG-Rec uses next-item prediction examples derived from user histories. Each training example contains a padded history sequence and the next-item target. The history item IDs are mapped to frozen item embeddings before entering the query generator; the target ID is used either to index the target embedding for MSE/InfoNCE or to index the full-catalog label for CE. A batch therefore has the form
\[
(\mathbf{H}, \mathbf{y}) =
\{([i_{t-L},\ldots,i_{t-1}], i_t)\}_{b=1}^{B},
\]
where $L$ is the maximum sequence length and $B$ is the batch size. Padding positions are masked and never treated as candidate items. This dataloader is intentionally the same sequential next-item protocol used by SASRec, so the comparison isolates the output interface rather than changing the recommendation task.

\subsection{Continuous Query Training Objectives}
\label{app:cqg_losses}

The MSE variant regresses the generated query $q_u$ to the frozen target embedding $e_{i^\star}$:
\[
\mathcal{L}_{\mathrm{MSE}} =
\|q_u - e_{i^\star}\|_2^2.
\]
The InfoNCE variant treats other targets in the batch as negatives:
\[
\mathcal{L}_{\mathrm{NCE}} =
-\log
\frac{\exp(q_u^\top e_{i^\star}/\tau)}
{\sum_{j\in\mathcal{B}}\exp(q_u^\top e_j/\tau)}.
\]
The CE variant scores the full frozen catalog:
\[
\mathcal{L}_{\mathrm{CE}} =
-\log
\frac{\exp(q_u^\top e_{i^\star}/\tau)}
{\sum_{j=1}^{N}\exp(q_u^\top e_j/\tau)}.
\]
All variants retrieve by dense similarity at inference time, so all encoded items remain reachable.

\section{SID Construction and Decoder Training}
\label{app:sid}

\subsection{Semantic-ID Construction}
\label{app:sid_construction}

The SID baseline follows the TIGER-style pipeline. First, a SASRec encoder is trained on the same chronological sequences used by CQG-Rec. Its learned item embedding matrix is frozen and clustered by hierarchical k-means. Each item receives a depth-3 code path. The resulting tuple of cluster indices is treated as the target token sequence for the autoregressive decoder. Code collisions are resolved by assigning residual tie-break identifiers when needed, but in the reported ML-1M run collisions are negligible: 99.4\% of items receive unique codes.

\subsection{TIGER-Style Dataloader}
\label{app:tiger_dataloader}

The SID decoder dataloader uses the same history-target pairs as CQG-Rec, but replaces the target item ID with its code tuple. Each batch contains a padded item-history sequence and a target SID sequence. Training minimizes token-level cross-entropy over the code positions. At inference, beam search expands partial SID prefixes under the code trie, maps completed codes back to item IDs, and returns the ranked decoded items.

\subsection{Loss Landscape}
\label{app:loss_landscape}

The CE objective benefits from catalog-complete visibility on smaller catalogs because each update contrasts the positive item against all alternatives. MSE is geometrically stable but lacks explicit repulsion from hard negatives. InfoNCE sits between these regimes and depends on the quality of in-batch negatives. This explains why CE obtains the strongest shallow Recall@10 in ML-1M, while MSE and BPR-like variants remain more scalable to larger catalogs.

\section{Dynamic Catalog Protocol}
\label{app:dynamic}

CQG-Rec naturally supports retraining-free item updates. A new item is encoded into the frozen item space and inserted into the ANN index. No output vocabulary, codebook, or trie needs to be rebuilt. In contrast, a SID system must assign the new item to a code path. If the assignment changes the code distribution or causes collisions, the codebook, trie, and decoder may need to be refreshed. In our planned dynamic-catalog evaluation, we hold out a fraction of items during training, insert them into the retrieval index only at test time, and measure new-item Recall@$K$, warm-item Recall@$K$, index update cost, and whether model retraining is required.
